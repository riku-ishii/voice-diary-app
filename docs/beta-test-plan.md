# ベータテスト計画書

「おやすみ前の1分」TestFlight ベータテスト計画

---

## 1. テスト概要

| 項目 | 内容 |
|------|------|
| テスト期間 | 2週間（目安） |
| テスト手段 | Apple TestFlight |
| テスター数 | 5〜10名 |
| フィードバック方法 | Googleフォーム + GitHub Issues |

---

## 2. テスト対象ユーザーの条件

### 必須条件

- iPhoneを所有していること（iOS 16以上）
- TestFlightアプリをインストールできること
- 毎晩1回、最低5日間テストに参加できること
- フィードバックフォームに回答できること

### 理想的なテスターの属性

- 20〜40代の社会人（メインターゲット層）
- 日記アプリや感情記録アプリの使用経験がある方
- メンタルケアや自己理解に関心がある方
- ITリテラシーが高くなくても参加可（一般ユーザー視点を重視）

### テスターの構成（目安）

| 属性 | 人数 | 目的 |
|------|------|------|
| 開発者本人 | 1名 | 技術的なバグの発見 |
| IT系の知人 | 2〜3名 | UX・パフォーマンスの検証 |
| 非IT系の知人 | 2〜3名 | 一般ユーザー視点での使いやすさ確認 |
| ターゲット層に近い方 | 2〜3名 | 実際の利用シーン（寝る前）での体験確認 |

---

## 3. テストシナリオ

### シナリオ1：初回起動と初めての記録

**目的**：初回ユーザーが迷わず使えるか確認

1. TestFlightからアプリをインストールする
2. アプリを起動する
3. マイク権限の許可ダイアログが表示されたら「許可」をタップ
4. ホーム画面が表示されることを確認
5. マイクボタンをタップして録音を開始する
6. 「今日は仕事が忙しかったです」など、自由に話す（30秒程度）
7. AIの返答が表示されることを確認
8. もう一度話す、またはAIの返答を聞く
9. 「話し終えた」ボタンをタップする
10. 感情カード（感情ラベル・色・一言まとめ）が表示されることを確認
11. ホーム画面に戻り、「今日はもう話したね」の表示を確認

**確認ポイント**：
- 操作に迷わなかったか
- AIの返答は自然だったか
- 感情カードの内容は話した内容と合っていたか

---

### シナリオ2：5日間の継続利用

**目的**：習慣として続けられるか・データ蓄積が正常か確認

1. 毎晩寝る前に1回、シナリオ1の手順で日記を記録する
2. 5日間連続で使用する
3. 各回、異なる内容（嬉しかったこと・疲れたこと・不安なことなど）を話す

**確認ポイント**：
- 毎日使いたいと思えたか
- 操作が面倒に感じなかったか
- 1分程度で終わる体感だったか

---

### シナリオ3：週間振り返りの確認

**目的**：週間データの可視化が正しく動作するか確認

1. 5日以上のデータが蓄積された状態で、下部タブの「振り返り」画面を開く
2. 色の丸が日ごとに表示されていることを確認
3. グラフが表示されていることを確認
4. 表示内容が実際に話した内容・感情と一致しているか確認

**確認ポイント**：
- 振り返り画面は直感的に理解できたか
- 色と感情の対応がわかりやすかったか
- 自分の1週間の心の変化を把握できたか

---

### シナリオ4：エラーケースの確認

**目的**：異常系の動作確認

1. 機内モード（オフライン）でアプリを起動し、録音を試みる
   - 適切なエラーメッセージが表示されるか
2. 録音中にアプリをバックグラウンドに移動する
   - 復帰後にアプリが正常に動作するか
3. 何も話さずに「話し終えた」をタップする
   - 適切に処理されるか
4. 非常に長い時間（3分以上）話し続ける
   - タイムアウトやクラッシュが発生しないか

## 4. フィードバック収集方法

### Googleフォーム（各テスターに毎日回答を依頼）

以下の項目を設定：

1. **日付**
2. **今日の使用で問題はありましたか？**（はい / いいえ）
3. **問題があった場合、詳細を教えてください**（自由記述）
4. **AIの返答は自然でしたか？**（5段階評価）
5. **感情カードの結果は納得できましたか？**（5段階評価）
6. **操作でわかりにくかった点はありますか？**（自由記述）
7. **その他、気づいたことや要望**（自由記述）

### 最終アンケート（テスト期間終了時）

1. **全体的な満足度**（5段階評価）
2. **リリースされたら使い続けたいですか？**（はい / どちらでもない / いいえ）
3. **人に薦めたいですか？**（NPS: 0〜10段階）
4. **最も良かった点**（自由記述）
5. **最も改善してほしい点**（自由記述）
6. **このアプリを一言で表すと？**（自由記述）

### バグ報告（随時）

- GitHub Issues に報告テンプレートを用意
  - 発生した画面
  - 操作手順
  - 期待した動作
  - 実際の動作
  - スクリーンショット（可能であれば）
  - iPhone機種・iOSバージョン

## 5. リリース判定基準

### 必須条件（すべて満たすこと）

| 項目 | 基準 |
|------|------|
| クラッシュ | テスト期間中のクラッシュがゼロ |
| コア機能 | 録音 → AI返答 → 感情カードの一連フローが全テスターで正常動作 |
| データ保存 | 感情データが正しく保存され、週間振り返りに反映される |
| エラーハンドリング | オフライン時・API障害時に適切なメッセージが表示される |

### 品質基準（目標値）

| 指標 | 目標 |
|------|------|
| AIの返答の自然さ（5段階） | 平均 3.5 以上 |
| 感情カードの納得度（5段階） | 平均 3.5 以上 |
| 全体満足度（5段階） | 平均 3.5 以上 |
| 継続利用意向 | 「はい」が60%以上 |
| NPS（推奨度） | 平均 6.0 以上 |

### リリース判定フロー

1. テスト期間終了後、全フィードバックを集計
2. 必須条件をすべて満たしているか確認
3. 品質基準の達成状況を確認
4. 未達の項目がある場合 → 改善後に追加テスト（1週間）
5. すべての基準を満たした場合 → App Store 審査提出へ

## 6. スケジュール（目安）

| フェーズ | 期間 | 内容 |
|---------|------|------|
| 準備 | 1〜2日 | TestFlightビルド作成、テスター招待、フォーム準備 |
| テスト前半 | 1週間 | 毎日の利用 + 日次フィードバック |
| 中間確認 | 1日 | フィードバック集計、致命的バグの修正 |
| テスト後半 | 1週間 | 改善版での継続テスト + 最終アンケート |
| 判定・改修 | 2〜3日 | リリース判定、最終修正 |
| 審査提出 | 1日 | App Store Connect で審査提出 |
